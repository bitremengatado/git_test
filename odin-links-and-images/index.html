This link will open a new tab

This anchor element not only had the "href" comment but also the "target" one. "Target" let's you dictate where the browser will open the new tab. You could write "_self", but the browser defaults to if if no other target is specified. In this case, we used the "_blank" target to redirect to a new, blank page.

Also worth noting. Although impossible to notice without opening console, this example also had a "rel" attribute. "rel" describes the relation between the current page and linked doc.

Here's the values added to "rel":
  • noopener:ensure the new tab cannot interact or access the original page. It adds security because, without it, the new page could use of JavaScript to manipulate yours.
  • noreferrer:prevents the new tab from knowing where the user is coming from ensuring privacy.

Theoretically, "noreferrer" also has "noopener" properties. By not knowing where the user is coming from (privacy) the new page also won't be able to use JavaScript to access it (security). However, using both is a good security measure because the new page could still manipulate the previous tab to change to a new webpage, tricking users.

Nowadays, browsers are so safety oriented they might still provide security if "target=_blank" is present, even if you forget the "rel" attribute. Still, always great to err in the side of caution and pair them up!

Absolute and Relative links

We've done a lot of absolute linking for now. That means we've used the exact path to reach another page. An absolute link looks like this scheme://domain/path. For example:

  • scheme: https
  • domain: theodinproject.com
  • path: about

That's not the only way to link something. Introducing: relative links.

Relative links don't need the domain to be reached. That's because it's used to link pages within the same website. Since both pages are in the same place, it's assumed the same domain is used.

Relative pages only include the path relative to the page you're creating the link on.

If the target page is in the same folder you could literally just tell the browser the "href" is the file itself. In our case it isn't. There's a folder called "pages" in the same folder as this page, index.html - inside the pages folder there's a file named "about.html" To reach it, let's use the location relative to the page we're in right now. that would be:

./pages/about.html

"./" tells the browser to look around the folder it already is.
"/pages" is where our link is located.
"about.html" is our link! Put it all together and it looks something a little like this:
